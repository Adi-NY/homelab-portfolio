# Trello — Real World Ticketing Experience
## Svdden Developments LTD (May 2023 — Nov 2025)

## Overview
During 2.5+ years at Svdden Developments, a managed services 
company operating FiveM server infrastructure for 50+ clients 
on Windows Server 2019/2025, I used Trello as the primary 
project management and ticketing platform across all client 
environments.

## How Trello Was Used
Each client had a dedicated Trello board with the following 
workflow stages:

- **To Do** — Incoming requests and reported issues
- **In Progress** — Actively being worked on
- **Need More Information** — Awaiting client clarification
- **Completed** — Resolved and documented
- **Not Possible** - Final Stage, contains an explanation of the reason.

## My Responsibilities
- Triaged and assigned incoming tickets across 7-8 active clients
- Documented technical findings and resolution steps in each card
- Managed 400+ completed tickets over the course of employment
- Escalated complex issues to senior developers with full context
- Maintained documentation standards across all boards

## Real World Example — Security Vulnerability Identification
While reviewing a plugin integration request, I independently 
identified a DDoS vulnerability caused by an unencrypted server IP 
broadcast communication. I flagged this in the ticket with a 
detailed technical explanation and escalated for a change request 
before deployment, preventing a potential security incident.

**Ticket comment documented:**
"The Laser Tag Plugin for the server has to be included with the 
backend compatibility module of the ESX Framework. This will 
require unencrypted communication and broadcast server IP 
information, which might cause security and Denial of Service 
issues (DDoS). A change request needs to be made."
